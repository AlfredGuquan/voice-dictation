# PRD — Voice Dictation

## 背景与目标

用说话替代打字的 Mac 语音输入法。解决两个核心场景：长文写作时打字速度是瓶颈，
边看内容边记笔记时手眼被占用无法输入。现有云端听写工具（Typeless）不稳定且有持续
订阅成本，macOS 内置听写缺乏智能清洗。本项目自建转录管道，用自选云端 ASR + 轻量
LLM 清洗，在可靠性和成本之间取得平衡。

---

### F1: 核心听写管道

- 动机: 打字是长文写作和边看边记场景下的输入瓶颈，需要一条从语音到干净文字的完整管道
- 描述: 热键 toggle 开始/结束听写。录音期间持续捕获麦克风音频，结束后发送至云端 ASR
        获取原始转录，再经轻量 LLM 清洗（去填充词"嗯啊哦"、去重复词，保留原意不重组），
        最终一次性注入当前系统焦点输入框的光标位置。无焦点输入框时文字存入历史记录并
        复制到剪贴板，发系统通知告知用户。网络失败时通知用户，音频保留在历史中可重试
- 约束:
  - Mac only
  - 中文为主，夹杂英文词（中英混合识别）
  - 只做轻量清洗——去填充词、去重复，不改原意不重组句子
  - 云端 ASR + 云端 LLM，供应商可替换
  - 批量注入（结束后一次性），非流式
- 验收标准:
  - 按热键开始录音，再按结束，清洗后文字出现在当前焦点输入框光标处
  - 中文语句中夹杂英文词能正确识别
  - 填充词（嗯、啊、哦）和重复词被去除，原意保留
  - Esc 键取消录音，不产生任何文字输出
  - Cmd+Z 能撤销刚注入的文字
  - 焦点不在输入框时，文字复制到剪贴板并弹出系统通知
  - 网络失败时弹出通知，音频保留在历史中可重试
- 依赖: 无

### F2: 浮动药丸 UI

- 动机: 用户需要明确的视觉反馈知道系统在录音、在处理、还是已完成
- 描述: 录音开始时屏幕底部浮出药丸形控件。录音状态下中间显示音频波形动画，
        左侧 ✕ 按钮取消，右侧 ✓ 按钮确认结束。用户按第二次热键或点击 ✓ 后，
        波形变为从左到右的进度条表示处理中。处理完成后药丸自动消失
- 约束:
  - 药丸悬浮于所有窗口之上，不遮挡主要内容
  - ✕ 按钮和 Esc 热键功能一致（取消录音）
  - ✓ 按钮和第二次热键功能一致（结束并处理）
- 验收标准:
  - 按热键后药丸从底部浮出，显示波形动画
  - 说话时波形有动态响应
  - 点击 ✕ 或按 Esc 取消录音，药丸消失
  - 点击 ✓ 或再按热键，波形变为进度条
  - 处理完成后药丸自动消失
- 依赖: F1

### F3: 个人词库

- 动机: ASR 对专有名词和领域术语识别不准，需要用户可维护的词库提升准确率
- 描述: 支持两种词库条目：识别词（需正确识别的专有名词，如"顾权""Claude Code"）
        和替换映射（说 X 写成 Y，如"CC"→"Claude Code"）。词库以文本文件存储，
        同时提供应用内 UI 进行增删改管理。词库在 LLM 清洗阶段应用
- 约束:
  - 文件存储作为 source of truth，UI 读写同一文件
  - 词库变更即时生效，不需重启应用
- 验收标准:
  - 在 UI 中添加识别词后，听写时该词能被正确识别
  - 在 UI 中添加替换映射后，听写中说出触发词，输出为替换后的文字
  - 直接编辑词库文件后，应用能识别变更并生效
  - 在 UI 中能浏览、搜索、编辑、删除词库条目
- 依赖: F1

### F4: 主窗口与历史记录

- 动机: 需要查看和复用过去的听写内容，以及对比清洗效果来调优词库
- 描述: 菜单栏常驻应用图标，点击可打开主窗口。主窗口包含历史听写列表（按时间倒序），
        支持全文搜索。每条记录保留原始转录和清洗后文字两个版本。提供独立对比视图，
        左右并排展示原始/清洗版本。主窗口同时作为设置入口（热键配置、词库管理）
- 约束:
  - 菜单栏图标常驻，主窗口按需打开
  - 历史记录本地持久化存储
- 验收标准:
  - 点击菜单栏图标能打开主窗口
  - 每次听写完成后，历史列表自动出现新记录
  - 搜索框输入关键词能过滤匹配的历史记录
  - 点击某条记录能看到清洗后文字，可一键复制
  - 切换到对比视图，左右并排显示原始转录和清洗后文字
  - 设置页面能配置热键和管理词库
- 依赖: F1
